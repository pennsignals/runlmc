runlmc.models.lmc module¶

class runlmc.models.lmc.LMC(Xs, Ys, normalize=True, kernels=None, lo=None, hi=None, m=None, name='lmc')[source]¶

Bases: runlmc.models.multigp.MultiGP

The main class of this package, LMC implements linearithmic Gaussian Process learning in the multi-output case. [TODO(PAPER)].

Upon construction, this class assumes ownership of its parameters and does not account for changes in their values.

The exact kernel that this approximates is the following:

\[K_{\text{exact}}=\sum_{q=1}^Q\left(A_qA_q^\top+ \boldsymbol\kappa_q I\right) \circ [k_q(X_i, X_j)]_{ij\in[D]^2} + \boldsymbol\epsilon I\]

\([\cdot]_{ij}\) represents a block matrix, with rows and columns possibly of different widths. \(\circ\) is the Hadamard product. \(\boldsymbol\epsilon I\) is a Gaussian noise addition, iid within each output. The input arrays for our observations of each of the different outputs are denoted \(X_i\) and may be variable-length. Each \(k_q(X_i,X_j)\) is built from a stationary Mercer kernel \(k_q\), where the \(ab\)-th entry of the rectangular matrix is \(k_q(\textbf{x}_a^{(i)}, \textbf{x}_b^{(j)})\) with \(\textbf{x}_a^{(i)}\) as the \(a\)-th input of the input set \(X_i\) (and correspondingly for \(j\)).

The \(\left(A_qA_q^\top+ \boldsymbol\kappa_q I\right)\) terms create a kernel which captures some linear correlation between outputs.

This class uses the SKI approximation, which shares a single grid \(U\) as the input array for all the outputs. Then, \(K_{\text{exact}}\) is interpolated from the approximation kernel \(K_{\text{SKI}}\), as directed in Thoughts on Massively Scalable Gaussian Processes by Wilson, Dann, and Nickisch. This is done with sparse interpolation matrices \(W\).

\[K_{\text{exact}}\approx K_{\text{SKI}} = W K W^\top + \boldsymbol\epsilon I\]

Above, \(K\) is a structured kernel over a grid \(U\), derived from \(A_q, k_q\) as before. The grid structure enables us to express \(K\) more succintly, relying on the Kronecker product \(\otimes\).

\[K=\sum_{q=1}^QA_qA_q^\top \otimes k_q(U, U)\]

Each \(A_q\) (only a column vector for now) is a parameter of this model, with name a<q>, where <q> is replaced with a specific number.

Parameters:
  • Xs – input observations, should be a list of numpy arrays, where the numpy arrays are one dimensional.
  • Ys – output observations, this must be a list of one-dimensional numpy arrays, matching up with the number of rows in Xs.
  • normalize – optional normalization for outputs Ys. Prediction will be un-normalized.
  • kernels – a list of (stationary) kernels which constitute the terms of the LMC sums prior to coregionalization. The \(q\)-th index here corresponds to \(k_q\) above. This list’s length is \(Q\)
  • lo – lexicographically smallest point in inducing point grid used (by default, a bit less than the minimum of input)
  • hi – lexicographically largest point in inducing point grid used (by default, a bit more than the maximum of input)
  • m – number of inducing points to use (by default, the total number of input points)
  • name (str) –
Raises:

ValueError if Xs and Ys lengths do not match.
Raises:

ValueError if normalization if any Ys have no variance or values in Xs have multiple identical values.

parameters_changed()[source]¶

This method is called automatically when linked parameters change, which the may during the optimization process.

Classes should update their posterior information, log likelihood, and gradients when this happens, such that self._raw_predict, self.log_likelihood, and self.gradient are consistent with the new parameters.
